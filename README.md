# End-to-End Clinical Data Pipeline (EDC → SDTM → ADaM → TLF) in SAS



### Objective



Build a reproducible clinical data pipeline to transform raw EDC data into SDTM and ADaM datasets and generate analysis outputs for efficacy and safety.



### Highlights



* Implemented end-to-end clinical data workflow
* Demonstrated understanding of SDTM vs ADaM structures
* Built analysis-ready datasets and generated TLF outputs



### Study Context

* Therapeutic Area: Diabetes
* Design: Single investigational product with multiple dose levels
* Treatment Groups: Low, Medium, High Dose
* Primary Endpoint: Change in systolic blood pressure (SBP)
* Safety: Adverse event summaries





### Datasets



#### SDTM:



* DM — Demographics
* AE — Adverse Events
* EX — Exposure
* VS — Vital Signs



#### ADaM:



* ADSL — Subject-level dataset
* ADVS — Vital signs analysis dataset



### Outputs



* Demographics summary by dose group
* Adverse event summary (safety)
* Mean change in SBP by visit and dose group
* Dose-response box plot



### Tools

* SAS (DATA step, PROC SQL, PROC MEANS, PROC SGPLOT)
* CDISC standards (SDTM, ADaM)
